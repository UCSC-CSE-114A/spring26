\subsection*{\textbf{Part 1: Lambda Calculus} (30 points)}

\question[6] A lambda calculus expression is in \emph{normal form} if it cannot be further reduced.  Evaluate the following lambda calculus expression to normal form using a series of $\beta$-reduction steps (and only $\beta$-reduction steps -- you shouldn't need anything else).  Start each line with \texttt{=b>}, as if you were using Elsa, and do just one $\beta$-reduction step per line.

Note: There may be multiple correct ways to reduce the expression.  A correct solution is any solution that Elsa would accept as correct.

\begin{verbatim}
(\x y -> (\f -> f x y)) (\r q -> q) (\s -> s) (\b c -> b)
\end{verbatim}
\begin{solutionorbox}[\stretch{1}]
Here is a possible answer (with comments for explanation). In each \verb|=b>| step, the chosen redex is the only available redex.  
\begin{verbatim}
  -- Reduce the redex `(\x y -> (\f -> f x y)) (\r q -> q)`
  =b> (\y -> (\f -> f (\r q -> q) y)) (\s -> s) (\b c -> b)
  -- Reduce the redex `(\y -> (\f -> f (\r q -> q) y)) (\s -> s)`
  =b> (\f -> f (\r q -> q) (\s -> s)) (\b c -> b)
  -- Reduce the redex `(\f -> f (\r q -> q) (\s -> s)) (\b c -> b)`
  =b> (\b c -> b) (\r q -> q) (\s -> s)
  -- Reduce the redex `(\b c -> b) (\r q -> q)`
  =b> (\c -> (\r q -> q)) (\s -> s)
  -- Reduce the redex `(\c -> (\r q -> q)) (\s -> s)`
  =b> \r q -> q
\end{verbatim}
\end{solutionorbox}

\newpage

The next question will involve a lambda calculus encoding of \textbf{lists}.  We can encode lists in lambda calculus using pairs, as follows:

\begin{verbatim}
let NIL   = \x -> TRUE
let CONS  = PAIR
let HEAD  = FST
let TAIL  = SND
let ISNIL = \lst -> lst (\h t -> FALSE)
\end{verbatim}

The list constructors are \texttt{NIL} and \texttt{CONS}.  \texttt{NIL} is the empty list, and a non-empty list consists of a pair of a list element and a list.

For instance, \texttt{CONS TWO (CONS THREE (CONS ONE NIL))} is the lambda calculus equivalent of the Haskell list \texttt{[2, 3, 1]}, which is just syntactic sugar for \texttt{2 : (3 : (1 : []))}.

The \texttt{ISNIL} function takes a list and returns \texttt{TRUE} if the list is \texttt{NIL} and \texttt{FALSE} otherwise.

You can (and should!) use the \texttt{NIL}, \texttt{CONS}, \texttt{HEAD}, \texttt{TAIL}, and \texttt{ISNIL} functions in your answers to the next question.

\subsection*{\textbf{Part 3: Working with Abstract Syntax Trees} (46 points)}

\noindent On section worksheet 5, we saw the following \verb|Expr| data type, which defines a grammar of abstract syntax trees for a tiny language of strings.  We will use this data type for most of the questions in this section.

\begin{verbatim}
data Expr = Var String    -- Variable
          | Str String    -- String literal
          | Cat Expr Expr -- Concatenate strings: `(++)`
          | Rev Expr      -- Reverse a string: `reverse`
\end{verbatim}

For example, we would could represent the expression \verb|(reverse (s ++ (reverse "cookie")))| with the following \verb|Expr|:

\begin{verbatim}
Rev (Cat (Var "s") (Rev (Str "cookie")))
\end{verbatim}

\question[8] Let us define the \emph{depth} of an \verb|Expr| as follows:

\begin{itemize}
\item The depth of a variable is 1.
\item The depth of a string literal expression is 1.
\item The depth of a concatenation expression \verb|Cat e1 e2| is 1 + the \emph{maximum} of the depths of \verb|e1| and \verb|e2|.
\item The depth of a reverse expression \verb|Rev e| is 1 + the depth of \verb|e|.
\end{itemize}

Define a Haskell function \verb|depth| that takes an \verb|Expr| and returns its depth as an \verb|Int|.  You can use the library functions in the Haskell reference at the end of the exam, but no other library functions. The type signature of \verb|depth| and one of the cases is provided for you below; fill in the rest of the definition.

\begin{verbatim}
depth :: Expr -> Int
depth (Var _) = 1
\end{verbatim}
\begin{solutionorbox}[\stretch{1}]
A simple implementation is:
  
\begin{verbatim}
depth (Str _) = 1
depth (Cat e1 e2) = 1 + max (depth e1) (depth e2)
depth (Rev e) = 1 + depth e
\end{verbatim}
\end{solutionorbox}

Finally, let's return to our tiny language of strings, and write an interpreter for it. Once again, here is the \verb|Expr| type that this section began with:

\begin{verbatim}
data Expr = Var String
          | Str String
          | Cat Expr Expr
          | Rev Expr
\end{verbatim}

Since \verb|Expr|s may contain variables, our interpreter will need to be an \emph{environment-passing} interpreter.  We will represent an environment as a list of pairs of variable names (which are \verb|String|s) and their values (which are also \verb|String|s, since every expression in this language evaluates to a string), using the following type alias:

\begin{verbatim}
type Env = [(String, String)]
\end{verbatim}

If an expression contains variables that are not bound in the environment, we won't be able to interpret it, so we'll use a \verb|Maybe| type as the return type of our interpreter.  If we try to evaluate an expression containing a variable that does not have a binding in the provided environment, our interpreter should return \verb|Nothing|.

Here are some example calls to \verb|eval|:

\begin{verbatim}
ghci> eval (Str "larry") []
Just "larry"

ghci> eval (Rev (Str "larry")) []
Just "yrral"

ghci> eval (Rev (Var "x")) [("x", "the creature")]
Just "erutaerc eht"

ghci> eval (Rev (Var "x")) [("y", "cali")]
Nothing

ghci> eval (Cat (Rev (Var "x")) (Rev (Var "y"))) [("x", "mo"), ("y", "mo")]
Just "omom"

ghci> eval (Cat (Rev (Var "x")) (Rev (Var "y"))) [("x", "cookie")]
Nothing

ghci> eval (Rev (Rev (Rev (Rev (Var "s"))))) [("s", "phoebe")]
Just "phoebe"
\end{verbatim}

Note: Although the above examples are simple, in general our interpreter should be able to handle arbitrarily deeply nested \verb|Expr|s.

\question[20] The type signature of our interpreter is provided below, with one case already written for you.  Fill in the remaining cases.  You can use library functions from the Haskell reference at the end of the exam, and you can also use the helper function at the bottom of the page.

\begin{verbatim}
eval :: Expr -> Env -> Maybe String
eval (Rev e) env = case eval e env of
  Just v -> Just (reverse v)
  _      -> Nothing
\end{verbatim}
\begin{solutionorbox}[5in]

\begin{verbatim}
eval (Var s) env = lookupInEnv s env
eval (Str s) _   = Just s
eval (Cat e1 e2) env = case (eval e1 env, eval e2 env) of
  (Just v1, Just v2) -> Just (v1 ++ v2)
  _                  -> Nothing
\end{verbatim}
\end{solutionorbox}

\begin{verbatim}
lookupInEnv :: String -> Env -> Maybe String
lookupInEnv _ []          = Nothing
lookupInEnv s ((k,v):kvs) = if s == k then Just v else lookupInEnv s kvs
\end{verbatim}

\begin{verbatim}
-- Church numerals
let ZERO  = \f x -> x
let ONE   = \f x -> f x

let HEAD  = FST
let TAIL  = SND
let ISNIL = \lst -> lst (\h t -> FALSE)

let ADD   = \n m -> n SUC m

let ISZ   = \n ->   -- (return TRUE if n == 0 and FALSE otherwise)
